# Docker Compose Stack Setup

## Portainer Install
- Resuorces- [Portainer CE install](https://docs.portainer.io/start/install-ce/server/docker/linux)
1. Create volume:
```bash

docker volume create portainer_data

```
2. Downlaod and install the server container:
```bash

docker run -d -p 8000:8000 -p 9443:9443 --name portainer --restart=always -v /var/run/docker.sock:/var/run/docker.sock -v portainer_data:/data portainer/portainer-ce:lts

```
- going to be on port 9443 - https
- Name: portainer
- Always has access to docker sock
- Going to have access to the data volume we have created already
- Pulls latest community edition

All pre done so now we can start configuration hehe

## Portainer Configuration
1. Find IP for container:
```bash

ip a
```
- grab container ip - most likely found under 2: eth0@if5

2. Get to portainer:
```bash

https:<IP>:9443
```
3. add username
4. Generate and store password
## Grab your portainer setup token(New addition to portainer)
- For me I was not used to this so I needed help finding it:
[How do I find, skip, or customize my portainer key[(https://docs.portainer.io/faqs/installing/setup-token)
1. In your media container:
```bash

docker ps

```
2. Grab portainer name/id
3. Open the logs based on how you are runnign docker for me it was:
```bash
docker logs <container id>
```
4. Shoudl see somthing like - "setup_token="
5. Copy that and paste it into the portainer login screen
6. hit create user

- I ran into the issue where it says the key I am using is invalid
- Ended up doing the following as I could not get any keys to work:

```bash

# 1. Stop and remove the current stuck Portainer container
docker stop portainer
docker rm portainer

# 2. Start it back up with the flag that disables the token requirement
docker run -d -p 9443:9443 --name portainer --restart always -v /var/run/docker.sock:/var/run/docker.sock -v portainer_data:/data portainer/portainer-ce:latest --no-setup-token
```

- ended up resolving issue later to ensure still secure
