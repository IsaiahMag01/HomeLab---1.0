# Pre-Build Configs
## What:
- Setting up our directories and making sure our tools are all aligned before installation of other tools


## Directory Configuration:
- Most of this is going to be done within the /data folder where our media and some applications will live
- I learned much of this configuration from TechHut and a numer of other sources and continue to use this directory Stack
### Setting up downloads folder:
1. Move into the data directory
```bash
cd /data
```
2. Create Directories
```bash
mkdir -p downloads/qbittorrent/{completed,incomplete,torrents} && mkdir -p downloads/nzbget/{completed,intermediate,nzb,queue,tmp}
```
Output: 
```txt
data
├── books
├── downloads
│   ├── qbittorrent
│   │   ├── completed
│   │   ├── incomplete
│   │   └── torrents
│   └── nzbget
│       ├── completed
│       ├── intermediate
│       ├── nzb
│       ├── queue
│       └── tmp
├── movies
├── music
├── shows
└── youtube
```
3. Make needed directories outside of downloads:
```bash
mkdir tv-shows && mkdir music && mkdir books
```
## Configure server so that our vpn client can reach the server
- Need to make come changes within our LXC Config
1. Go to LXC container config
```bash

/etc/pve/lxc/100.conf

```
2. Added the following configurations:
```bash

lxc.cgroup2.devices.allow: c 10:200 rwm
lxc.mount.entry: /dev/net dev/net none bind,create=dir
lxc.mount.entry: /dev/net/tun dev/net/tun none bind,create=file

```
- This allows our unprivialge container to access the device for our vpn network tunnel we will be creating
3. Reboot the server to commit changes to this configuration
4. Started media server again

## Portainer
- Resuorces- [Portainer CE install](https://docs.portainer.io/start/install-ce/server/docker/linux)
1. Create volume:
```bash

docker volume create portainer_data

```
2. Downlaod and install the server container:
```bash

docker run -d -p 8000:8000 -p 9443:9443 --name portainer --restart=always -v /var/run/docker.sock:/var/run/docker.sock -v portainer_data:/data portainer/portainer-ce:lts

```
- going to be on port 9443 - https
- Name: portainer
- Always has access to docker sock
- Going to have access to the data volume we have created already
- Pulls latest community edition

All pre done so now we can start configuration hehe
